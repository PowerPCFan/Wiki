Kiosk Error Screens

The basic error screens on a Redbox have different color backgrounds, indicating the problem at a basic level. Credits to @mordanthanus in the Discord for posting this info.

  • Black with white text – Kiosk is booting or updating. Black screens automatically clear themselves when completed, usually 5-15 minutes. Watchdog must be turned on in RBD. ·
  • Orange with white text – Kiosk is conducting a partial or full maintenance sync and will reboot to customer screen when complete. ·
  • Green with white text – The kiosk engine reports a database access error and places kiosk into maintenance mode. SOFTFILE0x errors. · 
  • Red with white text – Machine error alert. Generally requires human intervention but may fix itself during an overnight reboot.

Kiosk Error Codes

These error codes show on screen in maintenance mode. Credits to @mordanthanus in the Discord for posting this info.

= 0073CDRR =

- MS will always dispatch these due to an EMV error.

= 0099CDRR =

- Indication that the EMV has failed or been tampered with. Plan on bringing on with you. Alert Irruptions are not field repairable.

= 0072CDRR =

- EMV disconnected. If a remote reboot doesn’t fix it, a visit is needed.

= 0096KRNL =

- Kiosk Engine has put up the maintenance view directly

= 005PMXA =

- Put away max attempts exceeded

= 0128VNDR =

- Vend door error

= INITSYNC =

- Initial Sync is currently running

= 001ISTG =

- ItemStuckInGripper returned by HAL during vend or return

= 002OBDT =

- ObstructionDetected returned by HAL during vend or return

= 003CCFA =

- CameraCaptureFailure returned by HAL during vend or return

= 004MACH =

- MachineError returned by HAL during vend

= 006MVFA =

- MoveFailure returned by HAL during vend

= 0032RSVB =

- Reservation pickup could not find the reserved barcodes in stock

= RPCSSM =

- Session closed to show maintenance during a return

= 0048PPBE =

- Preposition Picker Basket Empty during checkout

= 0064SCBE =

- Shopping Cart Basket Empty during vend

= HALSVCNR =

- HAL Services are not running.

= HALSTS-043 =

- One of the following conditions are read: 

1) Vend Door Open 

2) Gripper Obstructed (Fingers not retracted, Track not Closed, Sensors 1 or 6 on) 

3) Motion Controller Communication error (unable to communicate to Arcus/Proteus) 

4) Door sensors aren’t properly closed, or someone opened the kiosk while picker or drum were moving.

= SOFTFILE_01 =

- The kiosk engine reports a database access error and places kiosk into maintenance mode.* Try restarting the kiosk engine, if error persists, send to Level 2 for possible software support escalation.

= SOFTFILE_02 =

- The kiosk engine reports a database access error and places kiosk into maintenance mode.* SOFTFILE_02 errors should be sent directly to Software support by Level 1

= SOFTFILE_03 =

- The kiosk engine reports a database access error and places kiosk into maintenance mode. Send to level 2 for possible software support escalation.
